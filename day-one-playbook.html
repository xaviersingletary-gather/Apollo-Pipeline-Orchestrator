2

~07:18 AM

For STRIKE accounts, read the personalized content before confirming. For NURTURE/QUALIFY, you can skip straight to enrollment.

STRIKE rule: Fit >=70 + Signal >=70. These are your top logos. The system requires your eyes on the email and script before it ships.
NURTURE/QUALIFY rule: Pre-trusted. Rubber-stamp enroll. No review needed.

The run log lives in the repo:

runs/2026-06-16-DHL-run.md

It contains per-contact breakdowns: email subject, body, call script, and LinkedIn connection note. You can open it in any markdown viewer or in VS Code.

3
You confirm enrollment in an interactive Claude session
~09:00 AM

This is not a Slack bot command. You open a fresh Claude session and say "enroll DHL" (or "approve DHL"). Claude then calls the Apollo MCP tools directly — adding contacts to the sequence and logging them to the watcher state.

What happens behind the scenes: Claude calls apollo_emailer_campaigns_search, apollo_email_accounts_index, then apollo_emailer_campaigns_add_contact_ids with status paused (STRIKE) or active (NURTURE/QUALIFY). It also writes each contact to state/active-contacts.jsonl for the reply watcher.

Example output from gate.py approve after Claude does the MCP work:

APPROVED + ENROLLED DHL (6 contacts). Mode=csv. Upload CSV to Apollo: runs/2026-06-16 DHL/apollo-import.csv
4
You import the CSV into Apollo
~09:05 AM

This is the only UI step. You import the generated CSV so Apollo maps per-contact custom fields (email body, script, connection note) onto each contact.
