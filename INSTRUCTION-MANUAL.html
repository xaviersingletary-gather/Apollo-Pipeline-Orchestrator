What This Is

The Apollo Pipeline Orchestrator is a self-driving outbound system for Gather AI strategic sales reps. Every weekday morning it picks the top-scored account from a ranked hopper, sources up to 20 ICP contacts, enriches verified emails, writes personalized first-touch content per contact, stages everything in Apollo, and reports back via Slack. The human rep only reviews and confirms send.

Philosophy: Automate everything that does not require human judgment. Gate everything that does. Nothing ships without an explicit confirm and activate. The rep stays the human on the calls, the LinkedIn videos, and the send button. The system handles the prep.

All scoring rules, voice rules, and cadence logic are in files, not in anyone's head. Clone the repo, run setup, fill your identity, and you are live.

Team Onboarding

Fast Path (Recommended)

The fastest way to get running is to let Claude handle the wiring. Paste the repo into a Cowork session, connect your tools, and hand over your account list.

Repo link: https://github.com/xaviersingletary-gather/Apollo-Pipeline-Orchestrator — copy this and paste it into Claude Cowork.

  1. Open Claude Cowork. Paste the repo contents into the session.
  2. Connect your stack. Add Apollo, Slack, Gmail, and Clay as available tools.
  3. Hand Claude your account list. Paste your target accounts (names, domains, any notes you have).
  4. Tell Claude: "Set up the Apollo Pipeline Orchestrator for me. Fill my rep identity and Apollo config, score these accounts into a hopper, and register the morning driver and weekly digest scheduled tasks."
  5. One-time Apollo UI step. Create the four custom fields in Apollo and wire them into your sequence template per hopper/APOLLO_SETUP.md. Claude will remind you.
  6. Bank approvals. Click Run now once on the apollo-morning-driver task in the Scheduled panel. Approve the tool prompts. Every future run is unattended.

That is it. The next morning Claude starts sourcing contacts, enriching emails, writing content, and Slacking you the staged batch.

The Gate System

Accounts are ranked on a Fit x Signal matrix. The gate decides how much human oversight each account gets before it ships.

STRIKE
Fit >= 70, Signal >= 70
Approve Gate
Prep + stage the CSV. Slack the rep. Enroll ONLY after explicit human yes. Protects the no-spam bar on top logos.
NURTURE
Fit >= 70, Signal < 70
Auto Gate
Pre-trusted. One-tap enroll. No content review required.
QUALIFY
Fit 40-69, Signal >= 40
Auto Gate
Pre-trusted. One-tap enroll. No content review required.
KILL / IGNORE
Either < 40
Skip
Never worked. Off the queue.

Status Lifecycle

Every account moves through states tracked in hopper/hopper.jsonl:

todo → in_progress → staged_for_approval → enrolled → done | parked

  • todo — in the queue, not yet worked
  • in_progress — Morning Driver is mid-run (prevents double-work on re-fire)
  • staged_for_approval — STRIKE only. CSV built, waiting on rep
  • enrolled — contacts logged to watcher state; ready for Apollo upload
  • parked / done — out of the active queue

Daily Workflow

Morning (System Runs)

At ~07:04 on weekdays, the Morning Driver fires unattended:

  1. Pick next account. Reads hopper.jsonl, returns top-ranked todo account. Skips KILL. Flags L-confidence accounts for research.
  2. Claim it. Locks the account to in_progress so a re-fire never double-works it.
  3. Research. Pulls existing research file (<14 days) or runs fresh account-research. Extracts triggers, sub-orgs, pain points.
  4. Find + enrich contacts. Calls Clay with ICP titles (VP/Director+ of Ops, Supply Chain, Logistics, Distribution, Fulfillment). Paginates up to 3 passes. Keeps only verified-email contacts.
  5. Generate content. Writes bespoke email (subject + body), 30-45s script (call opener + LinkedIn video), and <300-char connection note per contact.
  6. Write CSVs. Outputs apollo-upload.csv (run record) and apollo-import.csv (Apollo-ready custom fields).
  7. Stage. Marks account staged_for_approval (STRIKE) or queues for auto enrollment (NURTURE/QUALIFY).
  8. Write run log. Saves audit trail to runs/<date>-<account>-run.md.
  9. Slack report. ONE message covering all accounts worked: counts, gate tier, queue depth, confirm-enroll instruction.

Mid-Morning (Human Touch)

  1. Review Slack report. Check contact counts and gate tier.
  2. STRIKE review. Open the run log. Read email + script for each contact. Reply "enroll [account]" in Slack when satisfied.
  3. NURTURE/QUALIFY confirm. Rubber-stamp with the same Slack reply.
  4. Import CSV. In Apollo: Import → CSV → match on Email → map Gather_* columns to custom fields.
  5. Activate. Unpause the sequence. Day-1 email is manual (you send it).

Critical: Never activate a contact before the import populates their custom fields, or a blank email sends. Always import first, then activate.

Throughout the Day (Response Watcher)

Every 15 minutes the system polls Gmail for replies from active contacts:

  • Real replies → auto-fires lead scoring, produces handover doc, Slacks you the score + link
  • OOO replies → Slacks you with "needs guidance" tag. No scoring.
  • Wrong-person replies → Slacks you with "needs guidance" tag. No scoring.

Apollo One-Time Setup

Apollo custom fields and sequence wiring are done once in the Apollo UI. After that, each run is a 1-minute CSV import.

Step 1 — Create 4 Custom Fields (~3 min)

  1. Top-right gear icon → Settings
  2. Left nav → Custom Fields (under Data / General)
  3. Make sure object selector is Contacts/People
  4. Click + Add Custom Field for each:
Field Name (type exactly)HoldsType
Gather_Email_SubjectDay-1 email subjectText
Gather_Email_BodyDay-1 bespoke email bodyText (or Long Text)
Gather_Script30-45s script (call opener + LinkedIn video)Text
Gather_Connection_NoteLinkedIn connection note (<300 chars)Text

Step 2 — Wire the Sequence (~5 min)

Open your target sequence. Wire each step to the custom field variables via the { } picker. Never hand-type the tokens.

  • Day-1 email (position 1): Subject = Gather_Email_Subject, Body = Gather_Email_Body
  • Call steps (Day 1, 3, 8): Notes = Gather_Script
  • LinkedIn video step (Day 5/6): Notes = Gather_Script
  • LinkedIn connection (Day 1): Message = Gather_Connection_Note

Step 3 — Per-Run Import (1 minute)

Each run writes apollo-import.csv with headers matching the custom field names. In Apollo: Import → CSV → match on Email → map each Gather_* column to its field. Run. Then review and activate.

Response Watcher

The pull-side companion to the Morning Driver. Polls Gmail every 15 minutes for replies from contacts enrolled in active sequences.

State Files

FilePurpose
state/active-contacts.jsonlAppended by Morning Driver on every enrollment. Lists who to watch.
state/processed-replies.jsonlAppended by Watcher. Prevents re-processing the same reply.

Classifier Rules

ClassificationTriggersAction
Real Any substantive reply: "tell me more," "not interested," "send info," questions, objections. Auto-fires lead scoring, produces handover doc, Slacks score + link.
OOO "Out of office," "on vacation," "will be back on [date]," auto-reply formatting. Slacks rep with reply text + "needs guidance" tag. No scoring.
Wrong Person "No longer with company," "wrong person," "remove me," "not my area." Slacks rep with reply text + "needs guidance" tag. No scoring.

Bias rule: When genuinely ambiguous, default to real and let the rep sort it. Under-classifying is worse than over-classifying.

Weekly Digest

Every Friday early afternoon, the system auto-generates a rollup and Slacks it to the rep:

  • Accounts worked this week
  • Contacts enrolled by gate tier
  • Replies by classification (real / OOO / wrong-person)
  • Hopper depth with REPLENISH flag when todo accounts drop below 3

When the REPLENISH flag fires, it is time to re-score or add new accounts to the hopper.

Troubleshooting & Gotchas

ProblemCauseFix
Unattended run does nothing / stalls Tool approvals not banked (Step 6) Click Run now once in Cowork Scheduled panel. Approve each tool prompt. Future runs fire unattended.
Emails send with blank body or literal tokens Custom field tokens hand-typed instead of inserted via { } picker Always insert merge variables via Apollo's variable button. Never type {{...}} by hand.
Blank email sends to contact Activated before CSV import populated custom fields Import CSV first. Verify preview renders correctly. Then activate.
Clay returns 0 contacts after 3 passes Account genuinely has no ICP contacts, or domain mismatch Slack notice fires automatically. Account returns to todo. Continue run.
Account stuck in "in_progress" Session crashed or was interrupted before releasing lock Run python3 hopper/gate.py set "<account>" todo to release.
Same reply scored twice processed-replies.jsonl not being written Check state file permissions. Watcher appends only; should never duplicate.
Sequence missing at enroll time Sequence deleted, renamed, or paused Verify apollo_config.json sequence_id matches live sequence in Apollo UI.

Safety Invariants

1. Account locked (in_progress) before work, released on any exit.

2. Unattended driver NEVER calls add_contact_ids or approve. Nothing sends without human confirm + activate.

3. All Apollo/Slack IDs come from live in-session tool calls; never fabricated.

4. processed-replies.jsonl + active-contacts.jsonl keep the watcher idempotent across 15-minute runs.
